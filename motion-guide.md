# モーションデザイン パーソナリティ解説

> このドキュメントは **motion-design-skill**（LottieFiles）をベースに、4つのモーションパーソナリティを人間が実務で使えるよう解説したものです。

---

## このドキュメントについて

モーションデザインには「なんとなく気持ちいい」「なんか重い」という感覚的な評価を超え、数値で定義できる体系があります。

motion-design-skill は LottieFiles が提供するエージェントスキルで、以下を体系化しています。

- **3つの柱**: 感情的意図 / 視覚的ナラティブ / モーションクラフト
- **4つのパーソナリティ**: 同じ UI でも全く異なる体験を生む定数セット
- **実装レシピ**: タイミング・イージング・コレオグラフィの具体的な数値

このサイトでは「同一コンテンツ × 同一デザイン × 異なるモーション」で、動きの違いだけが体験を変える事実を体感できます。

---

## 4つのパーソナリティ 一覧表

| パーソナリティ | 印象キーワード | Duration (fast/slow) | Easing | Overshoot |
|--------------|-------------|---------------------|--------|-----------|
| **Premium** | エレガント・リッチ・洗練 | 350ms / 750ms | `cubic-bezier(0.4,0,0.2,1)` | なし |
| **Energetic** | スポーティ・ダイナミック・速い | 100ms / 300ms | `cubic-bezier(0.16,1,0.3,1)` | 1.5 |
| **Playful** | ポップ・キュート・バウンシー | 180ms / 450ms | `cubic-bezier(0.34,1.56,0.64,1)` | 2.2 |
| **Corporate** | クリーン・ビジネス・信頼 | 250ms / 500ms | `cubic-bezier(0.2,0,0,1)` | なし |

## Premium — エレガント・リッチ

### どんな印象を与えるか

ゆったりとしたテンポで「品格」「余裕」を演出します。動きが滑らかすぎず、かつ機械的にも見えない絶妙な速度帯です。見る人に「高価なものを扱っている」という無意識の安心感を与えます。

### 向いているブランド・シーン

- ラグジュアリーブランド（アパレル、時計、宝飾）
- 高級不動産・ホテルサイト
- ファインダイニング
- プレミアム SaaS の landing page

### タイミング・イージングの特徴

```
Duration fast: 350ms  — ボタンホバーなど即時フィードバック
Duration slow: 750ms  — Hero入場、モーダル展開
Easing:        cubic-bezier(0.4, 0, 0.2, 1)  — Material Design の "standard" easing
Overshoot:     なし（0%）  — オーバーシュートは安っぽく見える
Stagger:       120ms 間隔  — カードは余裕を持って順に登場
Hero yFrom:    60px 下から上昇
```

**Enter > Exit の原則**: 登場は 750ms かけて丁寧に。退場は 350ms で素早く消える（ユーザーが待たされない）。

### ボタンホバー

ボタン自体は動かさない。背景フィル・矢印・テキスト色の 3 要素で差を表現する。

- **背景フィル**: `scaleX: 0 → 1` で左から横展開（`power2.inOut`、750ms）
- **矢印 + テキスト**: `x: +6` にゆっくりスライド、色はラベルと同タイミングでホワイトへ

### カードホバー

- **背景**: スローフェードイン（`power2.inOut`）
- **ボトムライン**: 左→右 横展開（`power2.inOut`）
- **アイコン**: `x: +4` にスライド

### 実装上のポイント

- **Scale 禁止**: カードの入場に scale は使わない。position + opacity のみ。大きさが変わると安っぽく見える。
- **GSAP での記述**:
  ```ts
  gsap.fromTo('.hero', { opacity: 0, y: 60 }, { opacity: 1, y: 0, duration: 0.75, ease: 'power2.inOut' });
  ```
- **Tailwind CSS クラス**:
  ```html
  <div class="transition-all duration-[350ms] ease-[var(--ease-premium)]">...</div>
  ```

## Energetic — スポーティ・ダイナミック

### どんな印象を与えるか

爆発的なスピードで「アドレナリン」「瞬発力」を伝えます。`expo.out` のイージングが持つ急激な減速感だけで十分な勢いが出る。squash/stretch などの複合効果は「ビクつき」に見えるため使わない。

### 向いているブランド・シーン

- スポーツブランド・フィットネスアプリ
- ゲーム・eスポーツ
- エナジードリンク・食品
- 若年層向け EC サイト

### タイミング・イージングの特徴

```
Duration fast: 100ms  — ほぼ瞬間（ボタン押下のレスポンス）
Duration slow: 300ms  — Hero入場でもこの速さ
Easing:        cubic-bezier(0.16, 1, 0.3, 1)  — expo.out 相当
Overshoot:     1.5 相当（back.out パラメータ）
Stagger:       40ms 間隔  — カードがほぼ同時に飛び込む
Hero yFrom:    -80px 上から飛んでくる
```

**シンプルさの原則**: `expo.out` の加速感だけで十分。`scaleY` のスクワッシュや `onComplete` での二次バウンスは「ビクつき」に見えるため禁止。速さ＋クリーンさがこのパーソナリティの核。

### ボタンホバー

- **背景フィル**: `opacity: 0 → 1` 瞬間フラッシュ（0.08s `expo.out`）
- **矢印 + テキスト**: `x: +8 → +4` にダートして収束（2段階 timeline）、色は即時ホワイトに切り替え

### カードホバー

- **背景**: 瞬間フラッシュ（0.1s `expo.out`）
- **ボトムライン**: 高速展開（`expo.out`）
- **アイコン**: `scale: 1 → 1.35 → 1` パルス（timeline）

### 実装上のポイント

- **Hero の入場**: Y:-80px から鋭く飛んでくる。`expo.out` のみ。scaleY は使わない。
- **GSAP での記述**:
  ```ts
  gsap.fromTo('.hero', { opacity: 0, y: -80 }, { opacity: 1, y: 0, duration: 0.3, ease: 'expo.out' });
  ```
- **Tailwind CSS クラス**:
  ```html
  <div class="transition-all duration-[100ms] ease-[var(--ease-energetic)]">...</div>
  ```

## Playful — ポップ・キュート

### どんな印象を与えるか

バウンスとオーバーシュートが「楽しさ」「遊び心」を表現します。Disney のスクワッシュ＆ストレッチ原則を UI に適用した感覚。子どもっぽさというより「イラスト的な生き生き感」です。

### 向いているブランド・シーン

- 子ども向けサービス・教育アプリ
- キャラクター商材・グッズ EC
- ソーシャル機能を持つアプリ
- イラスト系クリエイターのポートフォリオ

### タイミング・イージングの特徴

```
Duration fast: 180ms  — 素早いが Energetic より少し余裕がある
Duration slow: 450ms  — Hero入場も軽快
Easing:        cubic-bezier(0.34, 1.56, 0.64, 1)  — back.out(2.2) 相当
Overshoot:     2.2（back.out パラメータ）  — はっきり弾む
Stagger:       60ms 間隔  — カードが順番に跳ね上がる
Hero:          scale 0.65 + rotation -4° から登場
```

**Rotation**: Hero 入場に `rotation: -4deg → 0deg` を加える。小さな回転が「手書き感」「温かみ」を与える。カード入場も同様。

### ボタンホバー

- **背景フィル**: `opacity` フェードイン（fast 180ms）
- **矢印 + テキスト**: `x: +10` に `back.out(2.5)` でオーバーシュート、色はラベルと同タイミングでホワイトへ。離すと `back.out(1.8)` で戻る

### カードホバー

- **背景**: フェードイン
- **ボトムライン**: 展開（`back.out`）
- **アイコン**: `scale: 1.4` + `rotation: 15°`（`back.out(2.2)`）でポンと弾ける
- **タイトル**: `y: -3` に `back.out(2.0)` で浮き上がり

### 実装上のポイント

- **Scale + bounce**: `scale: 0.65 → 1.0` + `back.out(2.2)` で「ポン！」と現れる。
- **カード rotation**: `rotation: 6 → 0` へのスタガードアニメーション。
- **GSAP での記述**:
  ```ts
  gsap.fromTo('.hero', { opacity: 0, scale: 0.65, rotation: -4 }, { opacity: 1, scale: 1, rotation: 0, duration: 0.45, ease: 'back.out(2.2)' });
  ```
- **Tailwind CSS クラス**:
  ```html
  <div class="transition-all duration-[180ms] ease-[var(--ease-playful)]">...</div>
  ```

## Corporate — ビジネス・クリーン

### どんな印象を与えるか

直線的で予測可能な動きが「信頼性」「プロフェッショナル」を演出します。ユーザーを驚かせない設計が優先され、動きは「存在を主張しない補佐役」です。

### 向いているブランド・シーン

- BtoB SaaS・エンタープライズツール
- 金融・保険・医療サービス
- 行政・公共機関のサイト
- コンサルティングファーム

### タイミング・イージングの特徴

```
Duration fast: 250ms  — 十分速いが瞬間的ではない
Duration slow: 500ms  — モーダル展開など
Easing:        cubic-bezier(0.2, 0, 0, 1)  — Material Design の "standard" 変形版
Overshoot:     なし（0%）
Stagger:       80ms 間隔  — カードはゆったり順に現れる
Hero xFrom:    -50px 横スライドで登場（Y移動なし）
```

**X方向の意味**: Hero の入場を Y ではなく X:-50px にすることで「横方向の直進性」を演出。縦の浮き上がりは感情的すぎてコーポレートに合わない。

### ボタンホバー

- **背景フィル**: `opacity` スローフェード（`power3.out`、500ms）
- **矢印 + テキスト**: `x: +4` のみ、ラベルと同タイミングでゆっくりホワイトへ切り替え

### カードホバー

- **背景**: スローフェードのみ（`power3.out`）
- **ボトムライン**: 展開（`power3.out`）
- アイコン・タイトルは動かさない

### 実装上のポイント

- **移動は X のみ**: Hero は Y 移動なし。X:-50px からの横スライド。
- **カードは opacity のみ**: 入場時も scale なし、y なし。最もシンプル。
- **GSAP での記述**:
  ```ts
  gsap.fromTo('.hero', { opacity: 0, x: -50 }, { opacity: 1, x: 0, duration: 0.5, ease: 'power3.out' });
  ```
- **Tailwind CSS クラス**:
  ```html
  <div class="transition-all duration-[250ms] ease-[var(--ease-corporate)]">...</div>
  ```

## ボタンホバーの設計原則

ボタン自体の位置・サイズは変えない。ユーザビリティ上の問題（クリック領域のずれ、視覚的なぶれ）を避けるため、動かすのは **背景フィルと矢印アイコンのみ**。

テキスト（`.btn-label`）と矢印（`.btn-arrow`）は **常に同じ色でトランジション** する。二つが別の色になる瞬間があると視覚的な統一感が崩れる。

| | 背景フィル | 矢印 + テキスト |
|--|-----------|----------------|
| **Premium** | `scaleX: 0→1` 左から横展開（750ms） | `x: +6` ゆっくりスライド、同時に白へ |
| **Energetic** | `opacity` 瞬間フラッシュ（0.08s） | `x: +8→+4` ダートして収束、色は即時白 |
| **Playful** | `opacity` フェード | `x: +10` `back.out(2.5)` でオーバーシュート、色は同タイミングで白 |
| **Corporate** | `opacity` スローフェード（500ms） | `x: +4` のみ、色はゆっくり白 |

### コントラスト対応 — `textOnAccent`

CTAバナーのボタンはアクセントカラーを背景にしているため、背景色に応じた文字色を `textOnAccent` として `MotionConfig` に定義する。

| パーソナリティ | アクセントカラー | `textOnAccent` | 理由 |
|--------------|----------------|----------------|------|
| Premium | `#c9a84c`（ゴールド） | `var(--color-ink)`（濃色） | 明るい金色 → 暗い文字 |
| Energetic | `#e63946`（赤） | `#ffffff` | 彩度の高い赤 → 白テキスト |
| Playful | `#7209b7`（濃紫） | `#ffffff` | 暗い紫 → 白テキスト |
| Corporate | `#1d3557`（濃紺） | `#ffffff` | 最も暗い色 → 白テキスト必須 |

Hero のボタンは透明背景 + `var(--color-ink)` テキストで設計が異なるため、リセット色はバインド時に `btn.style.color` を読み取って個別に保持する。これにより同一の `bindButtonInteraction()` 関数で両ボタンに対応できる。

## スライダーアニメーション

スライダーのトラック切り替え（`.slider-track` の `xPercent`）にもパーソナリティを反映する。

| | 切り替え easing | 考え方 |
|--|---------------|--------|
| **Premium** | `power2.inOut`（750ms） | 両端が緩やかな S 字。余裕あるスクロール感 |
| **Energetic** | `expo.out`（fast × 2.5 ≒ 250ms） | 瞬間加速→急停止。勢いを画面全体で体感 |
| **Playful** | `power2.out`（450ms） | オーバーシュートなし。なめらかに減速して止まる |
| **Corporate** | `power3.out`（500ms） | 直線的な減速。迷いのない動き |

> **Playful にオーバーシュートを使わない理由**
>
> `back.out` によるオーバーシュートはボタンアイコンなど**小さい要素**では軽やかに見えるが、スライダートラックのような**画面幅に近い大きな要素**に適用すると、コンテンツが一瞬コンテナ外に飛び出してから戻ってくる動きになり、視覚的な違和感が強くなる。Playful の「弾む感じ」はカード・ボタン・アイコンで表現し、スライダーは `power2.out` でクリーンに止める。

セクション入場（スクロールトリガー）は以下の通り異なる軸で入場する。

| | 入場 |
|--|------|
| **Premium** | `y: +30` から上昇フェードイン |
| **Energetic** | `x: +60` から横スライドイン |
| **Playful** | `scale: 0.96` からズームイン |
| **Corporate** | `x: -40` から左スライドイン |

## カードリストアニメーション（「できること」セクション）

### 入場（スクロールトリガー）

各行がスタガードで順に入場する。

| | 方向 | 備考 |
|--|------|------|
| **Premium** | `y: +20` 上昇フェード | 縦方向の余韻 |
| **Energetic** | `x: +40` 右から飛び込む | 横の勢い |
| **Playful** | `x: +20` + `rotation: 1°` | 小さな傾きで生き生き感を追加 |
| **Corporate** | `x: -30` 左からスライド | Hero と同軸で一貫性を保つ |

### ホバー

各行の左端に縦アクセントライン（`.card-list-accent`）が伸びる。

| | 背景 | アクセントライン | タイトル |
|--|------|----------------|---------|
| **Premium** | ゆっくりフェードイン（`power2.inOut`） | ゆっくり上から下へ伸びる | 動かさない |
| **Energetic** | 瞬間フラッシュ（0.1s） | 高速展開（`expo.out`） | `x: +10` に素早くダート |
| **Playful** | フェードイン | 展開（`back.out`） | `x: +8` に `back.out(2.0)` でバウンド |
| **Corporate** | スローフェード | ゆっくり展開 | 動かさない |

## モーダルアニメーション

`.modal-overlay`（背景幕）と `.modal-content`（ダイアログ本体）を独立したタイミングで動かすことで奥行きと層感を生む。

### 開く

| | オーバーレイ | コンテンツ | アクセントライン |
|--|------------|-----------|----------------|
| **Premium** | ゆっくりフェードイン（750ms） | `y: +40` から上昇 | 0.2s 遅れで横展開 |
| **Energetic** | 高速フェード（0.15s） | `scale: 0.92→1` ズームイン | 0.05s 遅れで高速展開 |
| **Playful** | 素早くフェード | `scale: 0.78 + rotation: -4°` から `back.out(2.0)` でバウンド着地 | 0.1s 遅れで展開 |
| **Corporate** | スローフェード | `x: -50` から左スライドイン | 0.15s 遅れで展開 |

### 閉じる

開くアニメーションの逆方向、かつ **duration.fast** で素早く消える（ユーザーを待たせない）。

- Premium: `y: +20` に沈みながらフェードアウト
- Energetic: `scale: 0.94` に縮んで瞬間消滅
- Playful: `scale: 0.85 + rotation: +3°` で飛び去る
- Corporate: `x: -30` に滑り出してフェードアウト

モーダルは **オーバーレイクリック・閉じるボタン・Escape キー** の 3 経路で閉じられる。

## パーソナリティ選定チートシート

| 質問 | → 選択 |
|------|-------|
| ターゲットは意思決定者・管理職か？ | **Corporate** |
| 高価格帯・高品質を訴求したいか？ | **Premium** |
| 10–30代の若い層がメインか？ | **Energetic** or **Playful** |
| キャラクターやイラストが登場するか？ | **Playful** |
| スポーツ・フィットネス・ゲームか？ | **Energetic** |
| 「信頼」より「驚き」を優先したいか？ | **Energetic** or **Playful** |
| 一つのブランドサイト全体で使うなら？ | 一つのパーソナリティを選び **一貫させる** |

## よくある失敗パターンと修正方法

| 症状 | 原因 | 修正 |
|------|------|------|
| ロボット感がある | linear easing を使っている | ease-out / ease-in-out 系に変更 |
| のっぺりした印象 | 一次モーションしかない | shadow・opacity の二次モーションを追加 |
| 重い・遅い印象 | duration が長すぎる | duration 表を見て要素タイプに合わせる |
| うるさい・気が散る | 同時に動く要素が多すぎる | 1/3ルール: 同時に動かす要素は全体の1/3以下 |
| 個性がない | パーソナリティが混在している | プロジェクト全体で1つのパーソナリティに統一 |
| 安っぽく見える（高級志向なのに） | overshoot を使っている | Premium は overshoot = 0 厳守 |
| ビクついて見える（Energetic） | scaleY squash や onComplete バウンスを多用 | expo.out の速さだけで表現。複合効果は禁止 |

## このプロジェクトの実装構造

```
src/
├── motion/
│   ├── types.ts          ← MotionPersonality 型定義（textOnAccent フィールド含む）
│   ├── personalities.ts  ← 4パーソナリティの定数（真実の源）
│   └── motion-base.ts    ← GSAP 共通 timeline 関数
│       ├── heroTimeline()            入場アニメーション
│       ├── cardStagger()             フィーチャーカード入場コレオグラフィ
│       ├── ctaTimeline()             CTAバナー入場
│       ├── bindButtonInteraction()   ボタンホバー（背景・矢印・テキスト色を連動）
│       ├── bindCardInteraction()     フィーチャーカードホバー
│       ├── initSlider()              スライダー切り替え＋入場
│       ├── cardListEntrance()        カードリスト入場（内部関数）
│       ├── bindCardListInteraction() カードリストホバー
│       ├── initModal()               モーダル開閉
│       ├── initAmbient()             アンビエントレイヤー
│       └── initPage()                全体初期化エントリ
├── components/
│   ├── Hero.astro         ← MV・CTAボタン（透明背景 + インク色テキスト）
│   ├── FeatureCards.astro ← 特徴カード 3 枚
│   ├── Slider.astro       ← 証言スライダー 4 枚
│   ├── CardList.astro     ← 機能リスト 5 件（モーダルトリガー付き）
│   ├── CTABanner.astro    ← CTAバナー（アクセント色背景ボタン）
│   └── Modal.astro        ← 詳細モーダル（固定オーバーレイ）
└── styles/
    └── base.css           ← Tailwind v4 @theme（CSS 変数として公開）
```

各ページは `motionConfigs[personality]` から定数を取得するだけ。パーソナリティを変えたいときは `personality` prop を変更するだけで済みます。
